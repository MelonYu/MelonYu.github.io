Hexo+Github Pages 搭建个人博客 - 准备篇

Jun 6, 2017

引言

为什么写博客？

1. hexo介绍

Hexo官方网站中关于What is Hexo? 的介绍是这样的：

Hexo is a fast, simple and powerful blog framework. You write posts in Markdown (or other languages) and Hexo generates static files with a beautiful theme in seconds.

快速，简单且强大，三个词抓住了我的心。并且支持Markdown，这对于想专心码字的人来说，简直就是完美的搭配。

2. 准备

在安装Hexo之前，需要先安装Git以及node.js，并且，你需要有一个GitHub账号 。

我想，像我一样的学生应该都是没有钱买域名的，所以，直接用GitHub Pages就好了。

3. 安装Git

windows版Git下载地址：https://git-scm.com/download/win

下载完成后，双击安装就好了，安装过程中不需要配置什么。

安装完成后，打开cmd(命令提示符)，输入

git –version

看到具体的版本信息则证明Git已经成功安装。

后面我们将用到的是的Git Bash，使用命令行来进行操作。

4. 安装node.js

node.js下载地址：https://nodejs.org/en/download/

windows可以选择下载安装msi或者exe。也是跟着提示下一步就可以了。

安装完成后，打开cmd(命令提示符)，输入

node -v

看到具体的版本信息则证明node.js已经成功安装。后面我们就不需要再管他了。

5. 安装Hexo

在 Git 和 node.js 都成功安装之后，打开之前提到的Git Bash（双击图标或者在桌面右键选择Git Bash），依次输入以下命令（$ 是命令提示符，无需手动输入）：

$ npm install hexo-cli -g

npm默认为当前项目安装,npm在新版的node中就包含了。-g 表示全局安装。

安装完成之后，在电脑任意盘符下新建一个文件夹，一般命名为hexo(如：D:/hexo)，然后从Git Bash进入到这个文件夹：

$ cd D:/hexo

依次输入下列命令：

$ hexo init
$ hexo generate
$ hexo server

init初始化hexo到你指定的目录(D:/hexo)，generate自动根据当前目录下的文件，生成静态网页，server在本地运行服务。当出现下面一行时，说明本地服务运行成功：

INFO Hexo is running at http://localhost:4000/. Press Ctrl+C to stop.

此时，打开浏览器，输入http://localhost:4000 就可以看到博客的效果了。退出本地服务的方式为Ctrl+C。

这里顺便提一句，如果你的电脑上装了福昕阅读器，那么你将看不到初始博客效果，为什么呢？因为你的福昕阅读器占用了4000端口，我们只需要在启动服务器时更改端口就可以了：

$ hexo server -p 500

至此，HEXO安装完成，接下来，就是将你的博客通过GitHub Pages展示给全世界了。下一篇就是激动人心的时刻。
